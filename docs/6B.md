### 1. Database Migration Script
* [supabase_migration_6a2.sql](file:///a:/June2026/happy-soul/supabase_migration_6a2.sql):
  * **seed_metadata**: Table storing import records (`dataset_name`, `translator`, `commentary`, `version`, `chapters_count`, `verses_count`, `checksum`, `seeded_at`).

### 2. Seeding Engine Updates (`scripts/gita/`)
* [enrichTranslations.js](file:///a:/June2026/happy-soul/scripts/gita/enrichTranslations.js): Stage 3.
  * **Update Instead of Upsert**: Replaced bulk `.upsert` with batch `.update` statements executing concurrently inside `Promise.all` for each chunk of 50.
  * **Non-Null Security**: By updating only translation-specific columns (`translation`, `translation_author`, `translation_author_id`, `translation_language`) on rows mapped via `id`, the script avoids failing on PostgreSQL non-null constraints (like `api_id` or `chapter_id`).
* [enrichCommentaries.js](file:///a:/June2026/happy-soul/scripts/gita/enrichCommentaries.js): Stage 4.
  * **Update Instead of Upsert**: Similarly refactored bulk `.upsert` to batch `.update` statements executing concurrently in groups of 50.
  * **Non-Null Security**: Overwrites only commentary-specific columns (`commentary`, `commentary_author`, `commentary_author_id`, `commentary_language`) without transmitting required keys or complete verse definitions.
